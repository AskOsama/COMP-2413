1. SQL Overview

Definition: SQL

SQL (Structured Query Language) is the standard language for managing and manipulating relational databases. It allows you to create, modify, query, and control access to data stored in a relational database management system (RDBMS).

Characteristics of Relational DBMS

📊

Data in Tables

Data is organized into tables (relations) with rows and columns

🔗

Relationships

Tables can be related through common columns (keys)

📝

SQL Language

Standard language for all relational database operations

Why SQL?

SQL is a declarative language - you describe WHAT you want, not HOW to get it. The DBMS optimizes the execution plan automatically.

2. SQL Components

SQL is divided into three main categories based on the type of operations:

🏗️

DDL - Data Definition Language

(لغة تعريف البيانات)

Defines database structure

  • CREATE - Create objects
  • ALTER - Modify objects
  • DROP - Delete objects
  • RENAME - Rename objects
📝

DML - Data Manipulation Language

(لغة معالجة البيانات)

Works with data in tables

  • SELECT - Query data
  • INSERT - Add data
  • UPDATE - Modify data
  • DELETE - Remove data
🔐

DCL - Data Control Language

(لغة التحكم في البيانات)

Controls access permissions

  • GRANT - Give permissions
  • REVOKE - Remove permissions

This Lecture's Focus

In this lecture, we focus on DDL (Data Definition Language) - the commands used to create and manage database structures.

3. CREATE DATABASE

Before creating tables, you must first create a database to contain them.

Syntax

CREATE DATABASE database_name;

Example

-- Create a new database called University CREATE DATABASE University;

USE Statement

After creating a database, you must select it before creating tables:

-- Select the University database USE University;

Complete Example

-- Step 1: Create the database CREATE DATABASE University; -- Step 2: Select the database to use USE University; -- Now you can create tables in this database

4. CREATE TABLE

The CREATE TABLE statement defines a new table with its columns and data types.

Basic Syntax

CREATE TABLE table_name ( column1 datatype, column2 datatype, column3 datatype, ... );

Common Data Types

🔢

INTEGER / INT

Whole numbers

Age, Quantity, ID
🔤

VARCHAR(n)

Variable-length text up to n characters

Names, Emails, Addresses
📅

DATE

Date values (YYYY-MM-DD)

Birthdate, Hire Date
💰

DECIMAL(p,s)

Precise decimal numbers

Price, Salary, GPA

Example: Basic Table

-- Create a simple authors table CREATE TABLE authors ( author_id INTEGER, first_name VARCHAR(50), last_name VARCHAR(50), birth_date DATE );

Important Notes

  • Table and column names should be descriptive
  • Choose appropriate data types for each column
  • VARCHAR size should accommodate the maximum expected length

5. Primary Keys

Definition: Primary Key

A primary key is a column (or combination of columns) that uniquely identifies each row in a table. Primary key values must be unique and cannot be NULL.

Single Column Primary Key

-- Method 1: Inline definition CREATE TABLE Student ( Sid INTEGER PRIMARY KEY, sname VARCHAR(100), age INTEGER, sex VARCHAR(10) );
-- Method 2: Constraint at the end CREATE TABLE Student ( Sid INTEGER, sname VARCHAR(100), age INTEGER, sex VARCHAR(10), PRIMARY KEY (Sid) );

Composite Primary Key

When a single column cannot uniquely identify a row, you can use multiple columns together:

-- Composite primary key using two columns CREATE TABLE Marks ( Sid INTEGER, Cid INTEGER, marks INTEGER, PRIMARY KEY (Sid, Cid) );

Key Insight

In the Marks table, neither Sid nor Cid alone can uniquely identify a row (a student can have marks in multiple courses, and a course can have multiple students). Together, they form a unique combination.

6. Foreign Keys

Definition: Foreign Key

A foreign key is a column (or columns) that references the primary key of another table. It establishes a relationship between two tables and enforces referential integrity.

Syntax

CREATE TABLE table_name ( column1 datatype, column2 datatype, FOREIGN KEY (column_name) REFERENCES other_table(primary_key) );

Example: Course Table with Foreign Key

-- Course table references Professor table CREATE TABLE Course ( Cid INTEGER PRIMARY KEY, cname VARCHAR(100), Pid INTEGER, FOREIGN KEY (Pid) REFERENCES Professor(Pid) );

Referential Integrity

The DBMS ensures that foreign key values must exist in the referenced table (or be NULL if allowed).

Prevents Orphan Records

You cannot insert a course with a professor ID that doesn't exist in the Professor table.

7. University Database Schema

Let's build a complete database with four related tables:

Table Primary Key Foreign Keys Description
Student Sid - Student information
Professor Pid - Professor information
Course Cid Pid → Professor Course taught by professor
Marks Sid + Cid Sid → Student, Cid → Course Student grades in courses

Complete Schema SQL

-- 1. Create the Student table CREATE TABLE Student ( Sid INTEGER PRIMARY KEY, sname VARCHAR(100), age INTEGER, sex VARCHAR(10) ); -- 2. Create the Professor table CREATE TABLE Professor ( Pid INTEGER PRIMARY KEY, pname VARCHAR(100) ); -- 3. Create the Course table (references Professor) CREATE TABLE Course ( Cid INTEGER PRIMARY KEY, cname VARCHAR(100), Pid INTEGER, FOREIGN KEY (Pid) REFERENCES Professor(Pid) ); -- 4. Create the Marks table (references Student and Course) CREATE TABLE Marks ( Sid INTEGER, Cid INTEGER, marks INTEGER, PRIMARY KEY (Sid, Cid), FOREIGN KEY (Sid) REFERENCES Student(Sid), FOREIGN KEY (Cid) REFERENCES Course(Cid) );

Creation Order Matters!

Tables must be created in the correct order. You cannot reference a table that doesn't exist yet. In this example:

  1. Student and Professor (no dependencies)
  2. Course (depends on Professor)
  3. Marks (depends on Student and Course)

8. ALTER TABLE

The ALTER TABLE statement modifies an existing table's structure.

ADD Column

-- Add a new column to an existing table ALTER TABLE Student ADD email VARCHAR(100);

DROP Column

-- Remove a column from a table ALTER TABLE Student DROP COLUMN email;

Practical Example

-- Add phone number column to Professor table ALTER TABLE Professor ADD phone VARCHAR(20); -- Add department column to Professor table ALTER TABLE Professor ADD department VARCHAR(50);

Caution

Dropping columns permanently removes data. Make sure to backup important data before modifying table structures.

9. DROP Statements

DROP statements permanently remove database objects.

DROP TABLE

-- Remove a table and all its data DROP TABLE table_name;

DROP DATABASE

-- Remove an entire database DROP DATABASE database_name;

Examples

-- Drop the Marks table DROP TABLE Marks; -- Drop the entire University database DROP DATABASE University;

Warning: Irreversible Action!

  • DROP permanently deletes the object and all its data
  • There is no "undo" for DROP commands
  • You cannot drop a table that is referenced by foreign keys in other tables
  • Always ensure you have backups before dropping objects

10. RENAME TABLE

The RENAME statement changes the name of an existing table.

Syntax

RENAME TABLE old_name TO new_name;

Example

-- Rename Student table to Students RENAME TABLE Student TO Students; -- Rename Professor table to Faculty RENAME TABLE Professor TO Faculty;

Note

When renaming tables, remember to update any queries, views, or application code that references the old table name.

Summary and Key Takeaways

What We've Learned

  1. SQL Components: SQL consists of DDL (structure), DML (data), and DCL (permissions).
  2. CREATE DATABASE/TABLE: Used to define new databases and tables with their columns and data types.
  3. Data Types: Common types include INTEGER, VARCHAR, DATE, and DECIMAL for storing different kinds of data.
  4. Primary Keys: Uniquely identify rows; can be single column or composite (multiple columns).
  5. Foreign Keys: Create relationships between tables and enforce referential integrity.
  6. ALTER TABLE: Modify existing tables by adding or dropping columns.
  7. DROP/RENAME: Remove or rename database objects (use with caution!).
