1. Aggregate Functions

What are Aggregate Functions?

Aggregate functions operate on a multiset of values of a column of a relation, and return a single value. SQL provides five built-in aggregate functions:

  • COUNT – number of values
  • AVG – average value
  • MIN – minimum value
  • MAX – maximum value
  • SUM – sum of values

1.1 COUNT

Example 1: Count the male students

SELECT COUNT(*) FROM Student WHERE sex = 'M';

Query Result

COUNT(*)
3

Example 2: Count female students who have age > 22

SELECT COUNT(*) FROM Student WHERE sex = 'F' AND age > 22;

Query Result

COUNT(*)
1

Example 3: Find the average marks for student id 2

AVG of Sid 2 = (17 + 35) / 2 = 26

SELECT AVG(Mark) FROM Marks WHERE Sid = 2;

GROUP BY and HAVING

GROUP BY groups rows that have the same values into summary rows. HAVING is used to filter groups (similar to WHERE, but for groups).

Example 5: Find the Sid of students who have average marks more than 25

SELECT Sid, AVG(Mark) FROM Marks GROUP BY (Sid) HAVING AVG(Mark) > 25;

2. Nested Subqueries

What are Nested Subqueries?

SQL provides a mechanism for the nesting of subqueries. A subquery is a SELECT-FROM-WHERE expression that is nested within another query. A common use of subqueries is to perform tests for set membership, set comparisons, and set cardinality.

Example 1: Find the Pid and the name of the professor who teaches "Programming2"

SELECT Pname FROM Professor WHERE Pid = (SELECT Pid FROM Course WHERE Cname = 'Programming2');

Query Result

PidPname
3Dr. Ameera

= vs IN with Subqueries

Use = when the subquery returns a single value. Use IN when the subquery returns multiple values.

Example 2: Find the name of the courses taught by "Dr. Ameera" or "Dr. Fahd"

SELECT Cname, Pid FROM Course WHERE Pid IN (SELECT Pid FROM Professor WHERE Pname = 'Dr. Ameera' OR Pname = 'Dr. Fahd');

Query Result

CnamePid
Programming 23
Programming12
Statistic3
Visual Programming2
Internet Programming3

3. Calculated Columns

What are Calculated Columns?

Calculated columns represent derived attributes. They may not be stored in the database and are calculated when needed. In the SELECT command, we use math operators such as +, -, or * to create calculated columns.

Why "Expr1001"?

Calculated columns appear in results with names like Expr1001 because they are new derived columns that are not saved in the database. You can use AS to give them meaningful names (aliases).

SELECT Sid, Cid, Mark + 5 FROM Marks;

Summary and Key Takeaways

What We've Learned

  1. Aggregate Functions: SQL provides five aggregate functions – COUNT, AVG, MIN, MAX, and SUM – that operate on a set of values and return a single result.
  2. GROUP BY: Groups rows with the same values into summary rows, allowing aggregate functions to be applied per group.
  3. HAVING: Filters groups after aggregation, similar to how WHERE filters individual rows.
  4. Nested Subqueries: A SELECT-FROM-WHERE expression nested within another query. Use = for single-value results and IN for multiple-value results.
  5. Calculated Columns: Derived attributes created using arithmetic operators (+, -, *) in the SELECT clause. These are not stored in the database and appear with auto-generated names like Expr1001.
