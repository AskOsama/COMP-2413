2. DELETE Statement

Definition: DELETE

The DELETE statement removes one or more rows from a table. When combined with a WHERE clause, only the rows that match the condition are deleted.

Syntax

DELETE FROM table_name WHERE condition;

Important Rules

  • The WHERE clause specifies which rows to delete
  • Without a WHERE clause, ALL rows in the table are deleted
  • DELETE removes entire rows, not individual column values
  • The table structure remains intact after DELETE
  • Always double-check your WHERE condition before executing

3. DELETE Examples

Example 1: Delete a Student by Name

Write the SQL command to delete the student named "Wafa Alothman":

-- Delete the student named 'Wafa Alothman' DELETE FROM student WHERE s_name = 'Wafa Alothman';

Table Before DELETE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Wafa Alothman 21 F

Table After DELETE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F

Example 2: Delete by Condition

Delete all students whose age is 21:

-- Delete all students aged 21 DELETE FROM student WHERE age = 21;

Caution: DELETE Without WHERE

If you omit the WHERE clause, all rows in the table will be deleted!

-- DANGER: This deletes ALL rows from the table! DELETE FROM student;

The table still exists (its structure is preserved), but it will be completely empty. This is different from DROP TABLE, which removes the table entirely.

DELETE vs DROP

DELETE FROM table_name removes all data rows but keeps the table structure. DROP TABLE table_name removes the entire table (structure + data). Use DELETE when you want to clear data; use DROP when you want to remove the table completely.

4. UPDATE — All Rows

Definition: UPDATE

The UPDATE statement modifies existing data in a table. It uses the SET clause to specify which columns to change and what their new values should be.

Form 1: Update All Rows (No WHERE)

UPDATE table_name SET column1 = value1, column2 = value2;

Example: Set All Names to 'Fatima'

-- Update ALL student names to 'Fatima' UPDATE student SET s_name = 'Fatima';

Table Before UPDATE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Wafa Alothman 21 F

Table After UPDATE (All Rows)

S_id s_name age sex
4439601234 Fatima 21 F
4439601235 Fatima 22 F
4439601236 Fatima 21 F

Be Careful!

Without a WHERE clause, every row in the table is updated. This is usually not the intended behavior. Always verify that you want to change all rows before omitting WHERE.

5. UPDATE — Specific Rows

Form 2: Update Specific Rows (With WHERE)

UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;

Example: Update Name Where Age = 21

-- Update name to 'Fatima' only for students aged 21 UPDATE student SET s_name = 'Fatima' WHERE age = 21;

Table Before UPDATE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Wafa Alothman 21 F

Table After UPDATE (Specific Rows)

S_id s_name age sex
4439601234 Fatima 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Fatima 21 F

UPDATE vs DELETE

UPDATE changes data but keeps the rows. DELETE removes entire rows. Both use WHERE to target specific rows — and both affect ALL rows when WHERE is omitted.

6. UPDATE — Multiple Columns

You can update more than one column in a single UPDATE statement by separating column assignments with commas.

Syntax

UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;

Example: Update Name and Age

-- Update name to 'Nora' and age to 23 for student with S_id '4439601235' UPDATE student SET s_name = 'Nora', age = 23 WHERE S_id = '4439601235';

Table After UPDATE (Multiple Columns)

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Nora 23 F
4439601236 Wafa Alothman 21 F

7. Data Control Language (DCL)

Definition: DCL

Data Control Language (DCL) includes commands such as GRANT and REVOKE which mainly deal with the rights, permissions, and other controls of the database system. These commands are used to control access to data in the database by granting or revoking permissions.

✅

GRANT

Gives specific privileges to users, allowing them to perform certain operations on database objects

🚫

REVOKE

Removes previously granted privileges from users, restricting their access to database objects

Why DCL Matters

In real-world databases, not every user should have full access. DCL allows database administrators to enforce the principle of least privilege — users get only the permissions they need to do their job. This is essential for database security.

8. GRANT Command

The GRANT command gives specific permissions to users or roles on database objects.

Syntax

GRANT permission1, permission2, ... ON object TO user;

Common Permissions

Permission Description
SELECT Permission to read/query data from a table
INSERT Permission to add new rows to a table
UPDATE Permission to modify existing data in a table
DELETE Permission to remove rows from a table
ALL PRIVILEGES All available permissions on the object

GRANT Examples

-- Grant SELECT permission on Student table to user 'ahmed' GRANT SELECT ON student TO ahmed; -- Grant INSERT and UPDATE permissions to user 'sara' GRANT INSERT, UPDATE ON student TO sara; -- Grant all permissions on Student table to user 'admin' GRANT ALL PRIVILEGES ON student TO admin;

9. REVOKE Command

The REVOKE command removes previously granted permissions from users or roles.

Syntax

REVOKE permission1, permission2, ... ON object FROM user;

REVOKE Examples

-- Revoke UPDATE permission from user 'sara' REVOKE UPDATE ON student FROM sara; -- Revoke all permissions from user 'ahmed' REVOKE ALL PRIVILEGES ON student FROM ahmed;

GRANT vs REVOKE

Think of GRANT and REVOKE as opposites:

  • GRANT ... TO user — gives permissions
  • REVOKE ... FROM user — takes away permissions

Notice: GRANT uses TO, while REVOKE uses FROM.

10. SQL Language Categories Summary

SQL commands are organized into three main categories. Here is a complete overview of what we have covered:

🏗️

DDL — Data Definition Language

Defines database structure: CREATE, ALTER, DROP, RENAME

📊

DML — Data Manipulation Language

Works with data: INSERT, SELECT, UPDATE, DELETE

🔐

DCL — Data Control Language

Controls access: GRANT, REVOKE

Category Purpose Commands Covered In

DML Manipulate data in tables INSERT, SELECT, UPDATE, DELETE Lectures 3 & 4

Summary and Key Takeaways

What We've Learned

  1. DELETE: Removes rows from a table using a WHERE condition. Without WHERE, all rows are deleted but the table structure remains.
  2. DELETE vs DROP: DELETE removes data rows only; DROP removes the entire table including its structure.
  3. UPDATE (All Rows): Without WHERE, UPDATE changes every row in the table. Use with caution.
  4. UPDATE (Specific Rows): With WHERE, only rows matching the condition are modified. You can update multiple columns in one statement.
  5. DCL (Data Control Language): GRANT gives permissions; REVOKE removes them. Essential for database security.
  6. SQL Categories: DDL defines structure, DML manipulates data, DCL controls access permissions.
