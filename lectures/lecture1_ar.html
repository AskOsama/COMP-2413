1. البيانات مقابل المعلومات

ما هي البيانات؟

تعريف: البيانات (Data)

البيانات هي حقائق خام - حقائق غير معالجة لم يتم تنظيمها أو تحليلها بعد. البيانات بمفردها ليس لها معنى كبير بدون سياق.

ما هي المعلومات؟

تعريف: المعلومات (Information)

المعلومات هي بيانات تمت معالجتها أو تنظيمها أو تقديمها في سياق ذي معنى. المعلومات مفيدة لاتخاذ القرارات.

مثال على تحويل البيانات

تحويل البيانات إلى معلومات

البيانات الخام المعلومات المعالجة

الفكرة الرئيسية

البيانات + السياق + المعالجة = المعلومات

نفس البيانات يمكن أن تصبح معلومات مختلفة حسب كيفية معالجتها والسياق المطبق.

2. مقدمة في قواعد البيانات

تعريف: قاعدة البيانات (Database)

قاعدة البيانات هي مجموعة من البيانات المترابطة المنظمة بطريقة تسمح بالتخزين والاسترجاع والإدارة بكفاءة. وهي تمثل جانباً من العالم الحقيقي (العالم المصغر).

مفاهيم قواعد البيانات الأساسية

👤

الكيان (Entity)

كائن مميز نخزن بيانات عنه. أمثلة: شخص، كتاب، طالب، منتج

📝

السجل (Record)

مجموعة من عناصر البيانات المترابطة عن مثيل كيان واحد. يسمى أيضاً صف أو tuple.

🏷️

السمة (Attribute)

خاصية أو صفة للكيان. أمثلة: الاسم، العمر، البريد الإلكتروني، السعر

تمثيل الجدول

رقم الطالب الاسم البريد الإلكتروني التخصص
1001 أحمد علي ahmed@email.com علوم الحاسب
1002 سارة محمد sara@email.com نظم المعلومات
1003 عمر حسن omar@email.com علوم الحاسب

الجدول = الكيان | الأعمدة = السمات | الصفوف = السجلات

مزايا نهج قواعد البيانات

1. تقليل تكرار البيانات

يزيل البيانات المكررة بتخزين المعلومات في مكان واحد

2. تحسين سلامة البيانات

يضمن دقة واتساق البيانات من خلال القيود

3. مشاركة البيانات

يمكن لعدة مستخدمين الوصول لنفس البيانات في وقت واحد

4. أمان البيانات

ضوابط الوصول تحمي المعلومات الحساسة

5. استقلالية البيانات

التطبيقات معزولة عن التغييرات الفيزيائية للبيانات

6. الوصول الفعال للبيانات

استعلامات محسنة وفهرسة لاسترجاع سريع

7. النسخ الاحتياطي والاستعادة

آليات مدمجة للحماية من فقدان البيانات

8. الوصول المتزامن

التعامل مع عدة مستخدمين دون تعارضات

9. تحسين اتخاذ القرارات

يوفر بيانات منظمة للتحليل والتقارير

10. فرض المعايير

يضمن تنسيقات بيانات واصطلاحات تسمية متسقة

التكاليف والمخاطر

  • التكلفة الأولية: نفقات البرمجيات والأجهزة والتدريب
  • التعقيد: يتطلب مديرين مهرة
  • عبء الأداء: قد يكون أبطأ للتطبيقات البسيطة
  • نقطة فشل واحدة: تعطل قاعدة البيانات يؤثر على جميع التطبيقات

قاموس البيانات

قاموس البيانات (أو مستودع البيانات الوصفية) يحتوي على معلومات حول هيكل قاعدة البيانات - أوصاف الجداول والأعمدة وأنواع البيانات والقيود والعلاقات.

مدير قاعدة البيانات (DBA)

مدير قاعدة البيانات مسؤول عن إدارة نظام قاعدة البيانات، بما في ذلك:

  • تصميم وتنفيذ قاعدة البيانات
  • الأمان والتحكم في الوصول
  • إجراءات النسخ الاحتياطي والاستعادة
  • مراقبة الأداء وضبطه
  • دعم المستخدمين والتدريب

نماذج إدخال البيانات

توفر النماذج واجهة سهلة الاستخدام لإدخال البيانات في قاعدة البيانات. يمكن أن تشمل:

  • التحقق من صحة الإدخال لضمان جودة البيانات
  • قوائم منسدلة للقيم المتسقة
  • حسابات تلقائية
  • نصوص مساعدة وتعليمات

3. نظم إدارة قواعد البيانات (DBMS)

السياق التاريخي

قبل قواعد البيانات، كانت البيانات تُخزن في ملفات مسطحة مع قيود كثيرة:

  • تكرار البيانات وعدم الاتساق
  • صعوبة الوصول للبيانات
  • عزل البيانات
  • مشاكل السلامة
  • عدم وجود وصول متزامن

تعريف: نظام إدارة قواعد البيانات (DBMS)

نظام إدارة قواعد البيانات هو برنامج يمكّن المستخدمين من إنشاء وصيانة والتحكم في الوصول لقواعد البيانات. يعمل كواجهة بين المستخدمين/التطبيقات والبيانات المخزنة.

وظائف DBMS الأساسية

الإنشاء (Create)

تعريف هيكل قاعدة البيانات والجداول والعلاقات والقيود

التخزين (Store)

إدراج وصيانة البيانات بطريقة منظمة

التنظيم (Organize)

هيكلة البيانات منطقياً مع الجداول والفهارس والعروض

الاسترجاع (Retrieve)

الاستعلام واستخراج البيانات بكفاءة باستخدام SQL

ملاحظة مهمة

مايكروسوفت إكسل ليس نظام DBMS! بينما يمكن لإكسل تخزين البيانات في صفوف وأعمدة، فإنه يفتقر للميزات الأساسية لنظام DBMS الحقيقي مثل فرض سلامة البيانات، والتحكم في الوصول المتزامن، وقدرات الاستعلام بـ SQL.

مقدمة في SQL

تعريف: SQL

SQL (لغة الاستعلام الهيكلية) هي اللغة المعيارية للتفاعل مع قواعد البيانات العلائقية. تتيح لك إنشاء البيانات والاستعلام عنها وتحديثها وإدارتها.

-- مثال على استعلام SQL SELECT Name, Email FROM Students WHERE Major = 'Computer Science';

تنقسم SQL إلى عدة فئات:

  • DDL (لغة تعريف البيانات): CREATE، ALTER، DROP
  • DML (لغة معالجة البيانات): SELECT، INSERT، UPDATE، DELETE
  • DCL (لغة التحكم في البيانات): GRANT، REVOKE

4. أهمية تصميم قواعد البيانات

تصميم قواعد البيانات هو عملية إنتاج نموذج بيانات مفصل لقاعدة البيانات. جودة التصميم تؤثر مباشرة على نجاح النظام.

التصميم الجيد

  • ينتج نتائج صحيحة
  • أداء استعلام سريع
  • سهل الصيانة والتعديل
  • تكرار بيانات أقل
  • يفرض سلامة البيانات

التصميم السيء

  • نتائج غير صحيحة أو غير متسقة
  • أداء بطيء
  • صعب التعديل
  • تكرار مفرط
  • شذوذات في البيانات

المبدأ الأساسي

قاعدة البيانات المصممة جيداً هي أساس أي نظام معلومات ناجح. الوقت المستثمر في التصميم الصحيح يُجزي عدة مرات أثناء التطوير والصيانة.

أسئلة المراجعة

اختبر فهمك

  1. ما الفرق بين البيانات والمعلومات؟ أعط مثالاً.
  2. عرّف المصطلحات التالية: الكيان، السجل، السمة
  3. اذكر خمس مزايا لاستخدام نهج قواعد البيانات.
  4. ما هو نظام DBMS وما هي وظائفه الأساسية؟
  5. لماذا لا يُعتبر إكسل نظام DBMS؟
  6. اشرح لماذا التصميم الجيد لقواعد البيانات مهم.

الملخص والنقاط الرئيسية

ما تعلمناه

  1. البيانات مقابل المعلومات: البيانات هي حقائق خام؛ المعلومات هي بيانات معالجة توفر معنى وسياقاً.
  2. أساسيات قواعد البيانات: قاعدة البيانات هي مجموعة منظمة من البيانات المترابطة، تتكون من كيانات (جداول) وسجلات (صفوف) وسمات (أعمدة).
  3. مزايا قواعد البيانات: يوفر نهج قواعد البيانات تقليل التكرار وتحسين السلامة وأمان أفضل ومشاركة بيانات فعالة.
  4. دور DBMS: توفر نظم إدارة قواعد البيانات أدوات لإنشاء وتخزين وتنظيم واسترجاع البيانات بكفاءة.
  5. مقدمة SQL: SQL هي اللغة المعيارية للعمل مع قواعد البيانات العلائقية.
  6. التصميم مهم: التصميم الجيد لقواعد البيانات حاسم لنجاح النظام، يؤثر على الأداء وقابلية الصيانة وجودة البيانات.

المحاضرة القادمة: لغة تعريف البيانات SQL (DDL) - سنتعلم كيفية إنشاء قواعد البيانات والجداول باستخدام أوامر SQL.
