أهداف التعلم

بنهاية هاتين المحاضرتين، ستكون قادراً على:

  • فهم دور DML في لغة SQL
  • إدراج سجلات جديدة في الجداول باستخدام INSERT INTO
  • استرجاع البيانات باستخدام SELECT (جميع الأعمدة وأعمدة محددة)
  • تصفية نتائج الاستعلام باستخدام SELECT مع شروط WHERE
  • حذف السجلات من الجداول باستخدام DELETE

1. نظرة عامة على لغة معالجة البيانات (DML)

تعريف: DML

لغة معالجة البيانات (DML) هي الجزء من SQL المستخدم لإضافة واسترجاع وتعديل وإزالة البيانات المخزنة في جداول قواعد البيانات. بينما تحدد DDL الهيكل، تعمل DML مع البيانات الفعلية داخل تلك الهياكل.

أوامر DML الرئيسية

➕

INSERT

إضافة صفوف بيانات جديدة إلى الجدول

🔍

SELECT

استرجاع وعرض البيانات من جدول أو أكثر

✏️

UPDATE

تعديل البيانات الموجودة في الجدول

🗑️

DELETE

إزالة صفوف البيانات من الجدول

مرجع أوامر DML

الأمر الغرض مثال
SELECT استرجاع البيانات من الجداول SELECT * FROM Student;
INSERT إضافة صفوف جديدة للجدول INSERT INTO Student VALUES (...);
UPDATE تعديل البيانات الموجودة UPDATE Student SET age = 22;
DELETE إزالة صفوف من الجدول DELETE FROM Student WHERE ...;

تركيز المحاضرة

في هاتين المحاضرتين، نركز على DML (لغة معالجة البيانات) — الأوامر المستخدمة للتعامل مع البيانات داخل الجداول — ونقدم DCL (لغة التحكم في البيانات) لإدارة صلاحيات الوصول.

2. INSERT INTO - الإدراج

عبارة INSERT INTO تضيف صفوف بيانات جديدة إلى جدول موجود.

الصيغة

INSERT INTO table_name VALUES (value1, value2, value3, ...);

قواعد مهمة

  • يجب أن تكون القيم مرتبة بنفس ترتيب الأعمدة في الجدول
  • القيم النصية يجب أن تكون محاطة بعلامات اقتباس مفردة ('text')
  • القيم الرقمية لا تحتاج علامات اقتباس
  • عدد القيم يجب أن يطابق عدد الأعمدة

هيكل جدول الطالب

تذكر جدول Student من المحاضرة 2:

-- جدول الطالب: Sid (مفتاح أساسي), sname, age, sex CREATE TABLE Student ( Sid INTEGER PRIMARY KEY, sname VARCHAR(100), age INTEGER, sex VARCHAR(10) );

أمثلة: إدراج البيانات

إدراج سجلات الطلاب

-- إدراج الطالبة الأولى INSERT INTO Student VALUES (1, 'Sarah Altalab', 20, 'F'); -- إدراج الطالبة الثانية INSERT INTO Student VALUES (2, 'Munira Alhssine', 21, 'F'); -- إدراج الطالبة الثالثة INSERT INTO Student VALUES (3, 'Haya Aldoussari', 21, 'F'); -- إدراج الطالبة الرابعة INSERT INTO Student VALUES (4, 'Wafa Alothman', 22, 'F');

الجدول الناتج

Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F
4 Wafa Alothman 22 F

الإدراج مع تحديد الأعمدة

يمكنك أيضاً تحديد الأعمدة التي تريد الإدراج فيها:

INSERT INTO Student (Sid, sname, age, sex) VALUES (5, 'Nora Alharbi', 20, 'F');

هذا مفيد عندما يحتوي الجدول على أعمدة ذات قيم افتراضية أو عندما تريد الإدراج في أعمدة محددة فقط.

3. SELECT - استرجاع البيانات

عبارة SELECT تسترجع البيانات من جدول أو أكثر. وهي أكثر أوامر SQL استخداماً.

الشكل 1: اختيار أعمدة محددة

SELECT column1, column2, ... FROM table_name;

مثال: اختيار الاسم والعمر

-- استرجاع أعمدة الاسم والعمر فقط SELECT sname, age FROM Student;
sname age
Sarah Altalab 20
Munira Alhssine 21
Haya Aldoussari 21
Wafa Alothman 22

الشكل 2: اختيار جميع الأعمدة

SELECT * FROM table_name;

مثال: اختيار جميع بيانات الطلاب

-- استرجاع جميع الأعمدة من جدول Student SELECT * FROM Student;
Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F
4 Wafa Alothman 22 F

الفرق الرئيسي

استخدم SELECT * لاسترجاع جميع الأعمدة، أو حدد أسماء الأعمدة لاسترجاع البيانات التي تحتاجها فقط. تحديد الأعمدة هو ممارسة أفضل عموماً لأنه يجعل استعلاماتك أكثر كفاءة وقابلية للقراءة.

4. SELECT مع WHERE

جملة WHERE تصفي الصفوف بناءً على شرط. فقط الصفوف التي تحقق الشرط يتم إرجاعها.

الصيغة: جميع الأعمدة مع شرط

SELECT * FROM table_name WHERE condition;

الصيغة: أعمدة محددة مع شرط

SELECT column1, column2 FROM table_name WHERE condition;

مثال: إيجاد الطالبات بعمر 21 (جميع الأعمدة)

-- استرجاع جميع البيانات للطالبات بعمر 21 SELECT * FROM Student WHERE age = 21;
Sid sname age sex
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F

مثال: إيجاد أسماء الطالبات بعمر 21 (أعمدة محددة)

-- استرجاع أسماء الطالبات بعمر 21 فقط SELECT sname FROM Student WHERE age = 21;
sname
Munira Alhssine
Haya Aldoussari

معاملات WHERE الشائعة

المعامل المعنى مثال
= يساوي WHERE age = 21
!= أو <> لا يساوي WHERE age != 21
> أكبر من WHERE age > 20
< أصغر من WHERE age < 22
>= أكبر من أو يساوي WHERE age >= 21
<= أصغر من أو يساوي WHERE age <= 21

5. DELETE - الحذف

عبارة DELETE تزيل صفوفاً من الجدول بناءً على شرط.

الصيغة

DELETE FROM table_name WHERE condition;

مثال: حذف طالبة بالاسم

-- حذف الطالبة المسماة 'Haya Aldoussari' DELETE FROM Student WHERE sname = 'Haya Aldoussari';

الجدول بعد DELETE

Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
4 Wafa Alothman 22 F

تحذير: DELETE بدون WHERE

إذا حذفت جملة WHERE، سيتم حذف جميع الصفوف في الجدول!

-- خطر: هذا يحذف جميع الصفوف من الجدول! DELETE FROM Student;

تحقق دائماً من شرط WHERE قبل تنفيذ عبارة DELETE.

6. UPDATE - تحديث جميع الصفوف

عبارة UPDATE تعدل البيانات الموجودة في الجدول. عند استخدامها بدون جملة WHERE، تحدث كل صف.

الصيغة

UPDATE table_name SET column_name = new_value;

مثال: تعيين جميع الأسماء إلى 'فاطمة'

-- تحديث جميع أسماء الطالبات إلى 'Fatima' UPDATE Student SET sname = 'Fatima';

الجدول بعد UPDATE (جميع الصفوف)

Sid sname age sex
1 Fatima 20 F
2 Fatima 21 F
3 Fatima 21 F
4 Fatima 22 F

كن حذراً!

بدون جملة WHERE، يتم تحديث كل صف في الجدول. هذا عادة ليس السلوك المقصود. تحقق دائماً من أنك تريد تغيير جميع الصفوف قبل حذف WHERE.

7. UPDATE - تحديث صفوف محددة

لتحديث صفوف معينة فقط، أضف جملة WHERE لتحديد الصفوف المراد تعديلها.

الصيغة

UPDATE table_name SET column_name = new_value WHERE condition;

مثال: تحديث الاسم حيث العمر = 21

-- تحديث الاسم إلى 'Fatima' فقط للطالبات بعمر 21 UPDATE Student SET sname = 'Fatima' WHERE age = 21;

الجدول قبل UPDATE

Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F
4 Wafa Alothman 22 F

الجدول بعد UPDATE (صفوف محددة)

Sid sname age sex
1 Sarah Altalab 20 F
2 Fatima 21 F
3 Fatima 21 F
4 Wafa Alothman 22 F

UPDATE مقابل DELETE

UPDATE يغير البيانات لكنه يبقي الصفوف. DELETE يزيل صفوفاً كاملة. كلاهما يستخدم WHERE لاستهداف صفوف محددة — وكلاهما يؤثر على جميع الصفوف عند حذف WHERE.

8. لغة التحكم في البيانات (DCL)

تعريف: DCL

لغة التحكم في البيانات (DCL) هي الجزء من SQL المستخدم للتحكم في الوصول إلى البيانات في قاعدة البيانات. أوامر DCL تدير صلاحيات المستخدمين والأمان.

✅

GRANT - منح الصلاحيات

يمنح صلاحيات محددة للمستخدمين أو الأدوار، مما يسمح لهم بتنفيذ عمليات معينة على كائنات قاعدة البيانات.

GRANT permission ON object TO user;
🚫

REVOKE - إلغاء الصلاحيات

يزيل الصلاحيات الممنوحة سابقاً من المستخدمين أو الأدوار، مما يقيد وصولهم إلى كائنات قاعدة البيانات.

REVOKE permission ON object FROM user;

أمثلة DCL

-- منح صلاحية SELECT على جدول Student للمستخدم 'ahmed' GRANT SELECT ON Student TO ahmed; -- منح صلاحيات INSERT و UPDATE على جدول Student للمستخدم 'sara' GRANT INSERT, UPDATE ON Student TO sara; -- إلغاء صلاحية UPDATE من المستخدم 'sara' REVOKE UPDATE ON Student FROM sara;

لماذا DCL مهمة

في قواعد البيانات الحقيقية، ليس كل مستخدم يجب أن يكون لديه وصول كامل. DCL تسمح لمسؤولي قواعد البيانات بتطبيق مبدأ الحد الأدنى من الصلاحيات — المستخدمون يحصلون فقط على الصلاحيات التي يحتاجونها لأداء عملهم.

اكتب عبارات SQL للمهام التالية

باستخدام جداول Student و Course و Marks من المحاضرة 2:

  2. أدرج مقرراً جديداً بـ Cid=101، الاسم='Database Programming'، Pid=1
  3. استرجع جميع البيانات من جدول Course
  4. استرجع أسماء وأعمار جميع الطلاب فقط
  5. أوجد جميع الطلاب الذين أعمارهم أكبر من 20
  6. حدّث عمر الطالب ذي Sid=1 إلى 21
  7. احذف الطالبة التي اسمها 'Wafa Alothman'
  8. امنح صلاحية SELECT على جدول Student لمستخدم اسمه 'admin'

أسئلة المراجعة

اختبر فهمك

  1. ماذا تعني DML وما هي أوامرها الأربعة الرئيسية؟
  2. ما الفرق بين DML و DDL؟
  3. لماذا يجب أن تتطابق القيم في عبارة INSERT مع ترتيب أعمدة الجدول؟
  4. ما الفرق بين SELECT * و SELECT column1, column2؟
  5. ماذا يحدث إذا نفذت عبارة DELETE بدون جملة WHERE؟
  6. ما الفرق بين UPDATE بدون WHERE و UPDATE مع WHERE؟
  7. ماذا تعني DCL وما هي أوامرها الرئيسيتان؟
  8. اشرح الفرق بين GRANT و REVOKE مع مثال.

الملخص والنقاط الرئيسية

ما تعلمناه

  1. نظرة عامة على DML: لغة معالجة البيانات تتضمن INSERT و SELECT و UPDATE و DELETE للتعامل مع البيانات في الجداول.
  2. INSERT INTO: تضيف صفوفاً جديدة للجدول. القيم يجب أن تتطابق مع ترتيب الأعمدة وأنواع البيانات.
  3. SELECT: تسترجع البيانات — استخدم * لجميع الأعمدة أو حدد أسماء أعمدة محددة.
  4. جملة WHERE: تصفي الصفوف بناءً على شروط. تُستخدم مع SELECT و UPDATE و DELETE.
  5. DELETE: تزيل صفوفاً من الجدول. بدون WHERE، تُحذف جميع الصفوف.
  6. UPDATE: تعدل البيانات الموجودة. بدون WHERE، تُحدث جميع الصفوف؛ مع WHERE، تتغير الصفوف المطابقة فقط.
  7. DCL: لغة التحكم في البيانات (GRANT و REVOKE) تدير صلاحيات الوصول لقواعد البيانات.

نظرة مستقبلية: في المحاضرة القادمة، سنستكشف استعلامات SQL بمزيد من العمق — سنتعلم الفرز مع ORDER BY، دوال التجميع (COUNT، SUM، AVG)، وتجميع البيانات مع GROUP BY.
