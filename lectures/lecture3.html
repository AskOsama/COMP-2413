1. Data Manipulation Language (DML) Overview

Definition: DML

Data Manipulation Language (DML) is the subset of SQL used to add, retrieve, modify, and remove data stored in database tables. While DDL defines the structure, DML works with the actual data inside those structures.

Main DML Commands

➕

INSERT

Add new rows of data into a table

🔍

SELECT

Retrieve and display data from one or more tables

✏️

UPDATE

Modify existing data in a table

🗑️

DELETE

Remove rows of data from a table

DML Commands Reference

Command Purpose Example
SELECT Retrieve data from tables SELECT * FROM Student;
INSERT Add new rows to a table INSERT INTO Student VALUES (...);
UPDATE Modify existing data UPDATE Student SET age = 22;
DELETE Remove rows from a table DELETE FROM Student WHERE ...;

Lecture Focus

In these lectures, we focus on DML (Data Manipulation Language) — the commands used to work with data inside tables — and introduce DCL (Data Control Language) for managing access permissions.

2. INSERT INTO

The INSERT INTO statement adds new rows of data into an existing table.

Syntax

INSERT INTO table_name VALUES (value1, value2, value3, ...);

Important Rules

  • The values must be listed in the same order as the columns in the table
  • String values must be enclosed in single quotes ('text')
  • Numeric values do not need quotes
  • The number of values must match the number of columns

Student Table Structure

Recall the Student table from Lecture 2:

-- Student table: Sid (PK), sname, age, sex CREATE TABLE Student ( Sid INTEGER PRIMARY KEY, sname VARCHAR(100), age INTEGER, sex VARCHAR(10) );

Examples: Inserting Data

Insert Student Records

-- Insert first student INSERT INTO Student VALUES (1, 'Sarah Altalab', 20, 'F'); -- Insert second student INSERT INTO Student VALUES (2, 'Munira Alhssine', 21, 'F'); -- Insert third student INSERT INTO Student VALUES (3, 'Haya Aldoussari', 21, 'F'); -- Insert fourth student INSERT INTO Student VALUES (4, 'Wafa Alothman', 22, 'F');

Resulting Table

Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F
4 Wafa Alothman 22 F

Column-Specific INSERT

You can also specify which columns to insert into:

INSERT INTO Student (Sid, sname, age, sex) VALUES (5, 'Nora Alharbi', 20, 'F');

This is useful when the table has columns with default values or when you want to insert into specific columns only.

3. SELECT — Retrieving Data

The SELECT statement retrieves data from one or more tables. It is the most commonly used SQL command.

Form 1: Select Specific Columns

SELECT column1, column2, ... FROM table_name;

Example: Select Name and Age

-- Retrieve only name and age columns SELECT sname, age FROM Student;
sname age
Sarah Altalab 20
Munira Alhssine 21
Haya Aldoussari 21
Wafa Alothman 22

Form 2: Select All Columns

SELECT * FROM table_name;

Example: Select All Student Data

-- Retrieve all columns from Student table SELECT * FROM Student;
Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F
4 Wafa Alothman 22 F

Key Difference

Use SELECT * to retrieve all columns, or list specific column names to retrieve only the data you need. Specifying columns is generally better practice as it makes your queries more efficient and readable.

4. SELECT with WHERE

The WHERE clause filters rows based on a condition. Only rows that satisfy the condition are returned.

Syntax: All Columns with Condition

SELECT * FROM table_name WHERE condition;

Syntax: Specific Columns with Condition

SELECT column1, column2 FROM table_name WHERE condition;

Example: Find Students Aged 21 (All Columns)

-- Retrieve all data for students aged 21 SELECT * FROM Student WHERE age = 21;
Sid sname age sex
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F

Example: Find Names of Students Aged 21 (Specific Columns)

-- Retrieve only names of students aged 21 SELECT sname FROM Student WHERE age = 21;
sname
Munira Alhssine
Haya Aldoussari

Common WHERE Operators

Operator Meaning Example
= Equal to WHERE age = 21
!= or <> Not equal to WHERE age != 21
> Greater than WHERE age > 20
< Less than WHERE age < 22
>= Greater than or equal WHERE age >= 21
<= Less than or equal WHERE age <= 21

5. DELETE

The DELETE statement removes rows from a table based on a condition.

Syntax

DELETE FROM table_name WHERE condition;

Example: Delete a Student by Name

-- Delete the student named 'Haya Aldoussari' DELETE FROM Student WHERE sname = 'Haya Aldoussari';

Table After DELETE

Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
4 Wafa Alothman 22 F

Caution: DELETE Without WHERE

If you omit the WHERE clause, all rows in the table will be deleted!

-- DANGER: This deletes ALL rows from the table! DELETE FROM Student;

Always double-check your WHERE condition before executing a DELETE statement.

6. UPDATE — All Rows

The UPDATE statement modifies existing data in a table. When used without a WHERE clause, it updates every row.

Syntax

UPDATE table_name SET column_name = new_value;

Example: Set All Names to 'Fatima'

-- Update ALL student names to 'Fatima' UPDATE Student SET sname = 'Fatima';

Table After UPDATE (All Rows)

Sid sname age sex
1 Fatima 20 F
2 Fatima 21 F
3 Fatima 21 F
4 Fatima 22 F

Be Careful!

Without a WHERE clause, every row in the table is updated. This is usually not the intended behavior. Always verify that you want to change all rows before omitting WHERE.

7. UPDATE — Specific Rows

To update only certain rows, add a WHERE clause to specify which rows to modify.

Syntax

UPDATE table_name SET column_name = new_value WHERE condition;

Example: Update Name Where Age = 21

-- Update name to 'Fatima' only for students aged 21 UPDATE Student SET sname = 'Fatima' WHERE age = 21;

Table Before UPDATE

Sid sname age sex
1 Sarah Altalab 20 F
2 Munira Alhssine 21 F
3 Haya Aldoussari 21 F
4 Wafa Alothman 22 F

Table After UPDATE (Specific Rows)

Sid sname age sex
1 Sarah Altalab 20 F
2 Fatima 21 F
3 Fatima 21 F
4 Wafa Alothman 22 F

UPDATE vs DELETE

UPDATE changes data but keeps the rows. DELETE removes entire rows. Both use WHERE to target specific rows — and both affect ALL rows when WHERE is omitted.

8. Data Control Language (DCL)

Definition: DCL

Data Control Language (DCL) is the subset of SQL used to control access to data in the database. DCL commands manage user permissions and security.

✅

GRANT

Gives specific privileges to users or roles, allowing them to perform certain operations on database objects.

GRANT permission ON object TO user;
🚫

REVOKE

Removes previously granted privileges from users or roles, restricting their access to database objects.

REVOKE permission ON object FROM user;

DCL Examples

-- Grant SELECT permission on Student table to user 'ahmed' GRANT SELECT ON Student TO ahmed; -- Grant INSERT and UPDATE permissions on Student table to user 'sara' GRANT INSERT, UPDATE ON Student TO sara; -- Revoke UPDATE permission from user 'sara' REVOKE UPDATE ON Student FROM sara;

Why DCL Matters

In real-world databases, not every user should have full access. DCL allows database administrators to enforce the principle of least privilege — users get only the permissions they need to do their job.

Summary and Key Takeaways

What We've Learned

  1. DML Overview: Data Manipulation Language includes INSERT, SELECT, UPDATE, and DELETE for working with data in tables.
  2. INSERT INTO: Adds new rows to a table. Values must match the column order and data types.
  3. SELECT: Retrieves data — use * for all columns or list specific column names.
  4. WHERE Clause: Filters rows based on conditions. Used with SELECT, UPDATE, and DELETE.
  5. DELETE: Removes rows from a table. Without WHERE, all rows are deleted.
  6. UPDATE: Modifies existing data. Without WHERE, all rows are updated; with WHERE, only matching rows change.
  7. DCL: Data Control Language (GRANT and REVOKE) manages database access permissions.
