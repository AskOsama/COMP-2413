1. نظرة عامة على SQL

تعريف: SQL

SQL (لغة الاستعلام الهيكلية) هي اللغة المعيارية لإدارة ومعالجة قواعد البيانات العلائقية. تتيح لك إنشاء وتعديل والاستعلام عن البيانات والتحكم في الوصول إلى البيانات المخزنة في نظام إدارة قواعد البيانات العلائقية (RDBMS).

خصائص أنظمة RDBMS

📊

البيانات في جداول

البيانات منظمة في جداول (علاقات) مع صفوف وأعمدة

🔗

العلاقات

يمكن ربط الجداول من خلال أعمدة مشتركة (مفاتيح)

📝

لغة SQL

لغة معيارية لجميع عمليات قواعد البيانات العلائقية

لماذا SQL؟

SQL هي لغة تصريحية - تصف ماذا تريد، وليس كيف تحصل عليه. نظام DBMS يحسّن خطة التنفيذ تلقائياً.

2. مكونات SQL

تنقسم SQL إلى ثلاث فئات رئيسية بناءً على نوع العمليات:

🏗️

DDL - لغة تعريف البيانات

(Data Definition Language)

تحدد هيكل قاعدة البيانات

  • CREATE - إنشاء الكائنات
  • ALTER - تعديل الكائنات
  • DROP - حذف الكائنات
  • RENAME - إعادة تسمية الكائنات
📝

DML - لغة معالجة البيانات

(Data Manipulation Language)

تعمل مع البيانات في الجداول

  • SELECT - استعلام البيانات
  • INSERT - إضافة البيانات
  • UPDATE - تعديل البيانات
  • DELETE - إزالة البيانات
🔐

DCL - لغة التحكم في البيانات

(Data Control Language)

تتحكم في صلاحيات الوصول

  • GRANT - منح الصلاحيات
  • REVOKE - إزالة الصلاحيات

تركيز هذه المحاضرة

في هذه المحاضرة، نركز على DDL (لغة تعريف البيانات) - الأوامر المستخدمة لإنشاء وإدارة هياكل قواعد البيانات.

3. إنشاء قاعدة البيانات CREATE DATABASE

قبل إنشاء الجداول، يجب أولاً إنشاء قاعدة بيانات لاحتوائها.

الصيغة

CREATE DATABASE database_name;

مثال

-- إنشاء قاعدة بيانات جديدة باسم University CREATE DATABASE University;

عبارة USE

بعد إنشاء قاعدة البيانات، يجب تحديدها قبل إنشاء الجداول:

-- تحديد قاعدة بيانات University USE University;

مثال كامل

-- الخطوة 1: إنشاء قاعدة البيانات CREATE DATABASE University; -- الخطوة 2: تحديد قاعدة البيانات للاستخدام USE University; -- الآن يمكنك إنشاء جداول في هذه القاعدة

4. إنشاء الجدول CREATE TABLE

عبارة CREATE TABLE تحدد جدولاً جديداً مع أعمدته وأنواع بياناته.

الصيغة الأساسية

CREATE TABLE table_name ( column1 datatype, column2 datatype, column3 datatype, ... );

أنواع البيانات الشائعة

🔢

INTEGER / INT

أعداد صحيحة

العمر، الكمية، المعرف
🔤

VARCHAR(n)

نص متغير الطول حتى n حرف

الأسماء، البريد، العناوين
📅

DATE

قيم التاريخ (YYYY-MM-DD)

تاريخ الميلاد، تاريخ التوظيف
💰

DECIMAL(p,s)

أرقام عشرية دقيقة

السعر، الراتب، المعدل

مثال: جدول بسيط

-- إنشاء جدول مؤلفين بسيط CREATE TABLE authors ( author_id INTEGER, first_name VARCHAR(50), last_name VARCHAR(50), birth_date DATE );

ملاحظات مهمة

  • يجب أن تكون أسماء الجداول والأعمدة وصفية
  • اختر أنواع البيانات المناسبة لكل عمود
  • حجم VARCHAR يجب أن يستوعب الطول الأقصى المتوقع

5. المفاتيح الأساسية

تعريف: المفتاح الأساسي (Primary Key)

المفتاح الأساسي هو عمود (أو مجموعة أعمدة) يحدد كل صف في الجدول بشكل فريد. قيم المفتاح الأساسي يجب أن تكون فريدة ولا يمكن أن تكون NULL.

مفتاح أساسي من عمود واحد

-- الطريقة 1: تعريف مضمن CREATE TABLE Student ( Sid INTEGER PRIMARY KEY, sname VARCHAR(100), age INTEGER, sex VARCHAR(10) );
-- الطريقة 2: قيد في النهاية CREATE TABLE Student ( Sid INTEGER, sname VARCHAR(100), age INTEGER, sex VARCHAR(10), PRIMARY KEY (Sid) );

المفتاح الأساسي المركب

عندما لا يستطيع عمود واحد تحديد الصف بشكل فريد، يمكنك استخدام عدة أعمدة معاً:

-- مفتاح أساسي مركب من عمودين CREATE TABLE Marks ( Sid INTEGER, Cid INTEGER, marks INTEGER, PRIMARY KEY (Sid, Cid) );

الفكرة الرئيسية

في جدول Marks، لا يستطيع Sid أو Cid بمفرده تحديد الصف بشكل فريد (الطالب قد يكون له درجات في عدة مقررات، والمقرر قد يكون له عدة طلاب). معاً، يشكلان تركيبة فريدة.

6. المفاتيح الخارجية

تعريف: المفتاح الخارجي (Foreign Key)

المفتاح الخارجي هو عمود (أو أعمدة) يشير إلى المفتاح الأساسي في جدول آخر. يُنشئ علاقة بين جدولين ويفرض سلامة المرجعية.

الصيغة

CREATE TABLE table_name ( column1 datatype, column2 datatype, FOREIGN KEY (column_name) REFERENCES other_table(primary_key) );

مثال: جدول المقرر مع مفتاح خارجي

-- جدول Course يشير إلى جدول Professor CREATE TABLE Course ( Cid INTEGER PRIMARY KEY, cname VARCHAR(100), Pid INTEGER, FOREIGN KEY (Pid) REFERENCES Professor(Pid) );

سلامة المرجعية

يضمن DBMS أن قيم المفتاح الخارجي يجب أن توجد في الجدول المُشار إليه (أو تكون NULL إذا سُمح).

يمنع السجلات اليتيمة

لا يمكنك إدراج مقرر بمعرف أستاذ غير موجود في جدول Professor.

استكشف العرض التوضيحي التفاعلي لمخطط الجامعة

7. مخطط قاعدة بيانات الجامعة

لنبني قاعدة بيانات كاملة مع أربعة جداول مترابطة:

الجدول المفتاح الأساسي المفاتيح الخارجية الوصف
Student Sid - معلومات الطالب
Professor Pid - معلومات الأستاذ
Course Cid Pid → Professor المقرر يدرسه أستاذ
Marks Sid + Cid Sid → Student, Cid → Course درجات الطلاب في المقررات

كود SQL الكامل للمخطط

-- 1. إنشاء جدول الطالب CREATE TABLE Student ( Sid INTEGER PRIMARY KEY, sname VARCHAR(100), age INTEGER, sex VARCHAR(10) ); -- 2. إنشاء جدول الأستاذ CREATE TABLE Professor ( Pid INTEGER PRIMARY KEY, pname VARCHAR(100) ); -- 3. إنشاء جدول المقرر (يشير إلى Professor) CREATE TABLE Course ( Cid INTEGER PRIMARY KEY, cname VARCHAR(100), Pid INTEGER, FOREIGN KEY (Pid) REFERENCES Professor(Pid) ); -- 4. إنشاء جدول الدرجات (يشير إلى Student و Course) CREATE TABLE Marks ( Sid INTEGER, Cid INTEGER, marks INTEGER, PRIMARY KEY (Sid, Cid), FOREIGN KEY (Sid) REFERENCES Student(Sid), FOREIGN KEY (Cid) REFERENCES Course(Cid) );

ترتيب الإنشاء مهم!

يجب إنشاء الجداول بالترتيب الصحيح. لا يمكنك الإشارة إلى جدول غير موجود بعد. في هذا المثال:

  1. Student و Professor (بدون اعتماديات)
  2. Course (يعتمد على Professor)
  3. Marks (يعتمد على Student و Course)
تصور علاقات مخطط الجامعة

8. تعديل الجدول ALTER TABLE

عبارة ALTER TABLE تعدل هيكل جدول موجود.

إضافة عمود ADD

-- إضافة عمود جديد لجدول موجود ALTER TABLE Student ADD email VARCHAR(100);

حذف عمود DROP

-- إزالة عمود من جدول ALTER TABLE Student DROP COLUMN email;

مثال عملي

-- إضافة عمود رقم الهاتف لجدول Professor ALTER TABLE Professor ADD phone VARCHAR(20); -- إضافة عمود القسم لجدول Professor ALTER TABLE Professor ADD department VARCHAR(50);

تحذير

حذف الأعمدة يزيل البيانات نهائياً. تأكد من عمل نسخة احتياطية للبيانات المهمة قبل تعديل هياكل الجداول.

9. عبارات الحذف DROP

عبارات DROP تزيل كائنات قاعدة البيانات نهائياً.

حذف جدول DROP TABLE

-- إزالة جدول وجميع بياناته DROP TABLE table_name;

حذف قاعدة البيانات DROP DATABASE

-- إزالة قاعدة بيانات كاملة DROP DATABASE database_name;

أمثلة

-- حذف جدول Marks DROP TABLE Marks; -- حذف قاعدة بيانات University كاملة DROP DATABASE University;

تحذير: إجراء لا رجعة فيه!

  • DROP يحذف الكائن وجميع بياناته نهائياً
  • لا يوجد "تراجع" لأوامر DROP
  • لا يمكنك حذف جدول مُشار إليه بمفاتيح خارجية في جداول أخرى
  • تأكد دائماً من وجود نسخ احتياطية قبل حذف الكائنات

10. إعادة تسمية الجدول RENAME

عبارة RENAME تغير اسم جدول موجود.

الصيغة

RENAME TABLE old_name TO new_name;

مثال

-- إعادة تسمية جدول Student إلى Students RENAME TABLE Student TO Students; -- إعادة تسمية جدول Professor إلى Faculty RENAME TABLE Professor TO Faculty;

ملاحظة

عند إعادة تسمية الجداول، تذكر تحديث أي استعلامات أو عروض أو كود تطبيق يشير إلى اسم الجدول القديم.

تمارين عملية

جرب هذه العبارات SQL

  1. أنشئ قاعدة بيانات باسم "Library"
  2. أنشئ جدول "Book" بأعمدة: BookID (PK)، Title، Author، PublishYear، ISBN
  3. أنشئ جدول "Member" بأعمدة: MemberID (PK)، Name، Email، JoinDate
  4. أنشئ جدول "Borrowing" لتتبع أي عضو استعار أي كتاب (PK مركب مع FKs)
  5. أضف عمود "Phone" لجدول Member
  6. أعد تسمية جدول "Book" إلى "Books"
تدرب على أوامر DDL في العرض التوضيحي التفاعلي

أسئلة المراجعة

اختبر فهمك

  1. ماذا تعني DDL وما الأوامر التي تتضمنها؟
  2. ما الفرق بين DDL و DML و DCL؟
  3. اشرح الغرض من المفتاح الأساسي. ما القيود التي يفرضها؟
  4. متى تستخدم مفتاح أساسي مركب؟ أعط مثالاً.
  5. ما هو المفتاح الخارجي وكيف يفرض سلامة المرجعية؟
  6. لماذا ترتيب إنشاء الجداول مهم عند استخدام المفاتيح الخارجية؟
  7. ما الفرق بين DROP TABLE و ALTER TABLE DROP COLUMN؟
  8. اكتب SQL لإنشاء جدول "Employee" بـ: EmpID (PK)، Name، DeptID (FK إلى Department)، Salary

الملخص والنقاط الرئيسية

ما تعلمناه

  1. مكونات SQL: تتكون SQL من DDL (الهيكل)، DML (البيانات)، و DCL (الصلاحيات).
  2. CREATE DATABASE/TABLE: تُستخدم لتعريف قواعد بيانات وجداول جديدة مع أعمدتها وأنواع بياناتها.
  3. أنواع البيانات: الأنواع الشائعة تشمل INTEGER، VARCHAR، DATE، و DECIMAL لتخزين أنواع مختلفة من البيانات.
  4. المفاتيح الأساسية: تحدد الصفوف بشكل فريد؛ يمكن أن تكون عمود واحد أو مركبة (عدة أعمدة).
  5. المفاتيح الخارجية: تُنشئ علاقات بين الجداول وتفرض سلامة المرجعية.
  6. ALTER TABLE: تعدل الجداول الموجودة بإضافة أو حذف الأعمدة.
  7. DROP/RENAME: تزيل أو تعيد تسمية كائنات قاعدة البيانات (استخدمها بحذر!).

المحاضرة القادمة: لغة معالجة البيانات SQL (DML) - سنتعلم كيفية INSERT وUPDATE وDELETE وSELECT البيانات من الجداول.
