2. عبارة DELETE - الحذف

تعريف: DELETE

عبارة DELETE تزيل صفاً واحداً أو أكثر من الجدول. عند استخدامها مع جملة WHERE، يتم حذف الصفوف التي تطابق الشرط فقط.

الصيغة العامة

DELETE FROM table_name WHERE condition;

قواعد مهمة

  • جملة WHERE تحدد أي الصفوف سيتم حذفها
  • بدون جملة WHERE، يتم حذف جميع الصفوف في الجدول
  • DELETE يزيل صفوفاً كاملة، وليس قيم أعمدة فردية
  • يبقى هيكل الجدول كما هو بعد DELETE
  • تحقق دائماً من شرط WHERE قبل التنفيذ

3. أمثلة على DELETE

مثال 1: حذف طالبة بالاسم

اكتب أمر SQL لحذف الطالبة "Wafa Alothman":

-- حذف الطالبة المسماة 'Wafa Alothman' DELETE FROM student WHERE s_name = 'Wafa Alothman';

الجدول قبل DELETE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Wafa Alothman 21 F

الجدول بعد DELETE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F

مثال 2: الحذف بشرط

حذف جميع الطالبات اللواتي عمرهن 21:

-- حذف جميع الطالبات بعمر 21 DELETE FROM student WHERE age = 21;

تحذير: DELETE بدون WHERE

إذا حذفت جملة WHERE، سيتم حذف جميع الصفوف في الجدول!

-- خطر: هذا يحذف جميع الصفوف من الجدول! DELETE FROM student;

الجدول لا يزال موجوداً (هيكله محفوظ)، لكنه سيكون فارغاً تماماً. هذا يختلف عن DROP TABLE الذي يزيل الجدول بالكامل.

DELETE مقابل DROP

DELETE FROM table_name يزيل صفوف البيانات فقط لكن يحتفظ بهيكل الجدول. DROP TABLE table_name يزيل الجدول بالكامل (الهيكل + البيانات). استخدم DELETE عندما تريد مسح البيانات؛ واستخدم DROP عندما تريد إزالة الجدول نهائياً.

4. UPDATE - تحديث جميع الصفوف

تعريف: UPDATE

عبارة UPDATE تعدل البيانات الموجودة في الجدول. تستخدم جملة SET لتحديد الأعمدة المراد تغييرها وقيمها الجديدة.

الصيغة 1: تحديث جميع الصفوف (بدون WHERE)

UPDATE table_name SET column1 = value1, column2 = value2;

مثال: تعيين جميع الأسماء إلى 'Fatima'

-- تحديث جميع أسماء الطالبات إلى 'Fatima' UPDATE student SET s_name = 'Fatima';

الجدول قبل UPDATE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Wafa Alothman 21 F

الجدول بعد UPDATE (جميع الصفوف)

S_id s_name age sex
4439601234 Fatima 21 F
4439601235 Fatima 22 F
4439601236 Fatima 21 F

انتبه!

بدون جملة WHERE، يتم تحديث كل صف في الجدول. هذا عادةً ليس السلوك المطلوب. تأكد دائماً من أنك تريد تغيير جميع الصفوف قبل حذف WHERE.

5. UPDATE - تحديث صفوف محددة

الصيغة 2: تحديث صفوف محددة (مع WHERE)

UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;

مثال: تحديث الاسم حيث العمر = 21

-- تحديث الاسم إلى 'Fatima' فقط للطالبات بعمر 21 UPDATE student SET s_name = 'Fatima' WHERE age = 21;

الجدول قبل UPDATE

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Wafa Alothman 21 F

الجدول بعد UPDATE (صفوف محددة)

S_id s_name age sex
4439601234 Fatima 21 F
4439601235 Haya Aldoussari 22 F
4439601236 Fatima 21 F

UPDATE مقابل DELETE

UPDATE يغير البيانات لكن يحتفظ بالصفوف. DELETE يزيل صفوفاً كاملة. كلاهما يستخدم WHERE لاستهداف صفوف محددة — وكلاهما يؤثر على جميع الصفوف عند حذف WHERE.

6. UPDATE - تحديث أعمدة متعددة

يمكنك تحديث أكثر من عمود في عبارة UPDATE واحدة بفصل تعيينات الأعمدة بفواصل.

الصيغة العامة

UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;

مثال: تحديث الاسم والعمر

-- تحديث الاسم إلى 'Nora' والعمر إلى 23 للطالبة ذات الرقم '4439601235' UPDATE student SET s_name = 'Nora', age = 23 WHERE S_id = '4439601235';

الجدول بعد UPDATE (أعمدة متعددة)

S_id s_name age sex
4439601234 Sarah Altalab 21 F
4439601235 Nora 23 F
4439601236 Wafa Alothman 21 F

مرجع أوامر DML

الأمر الغرض الصيغة
INSERT إضافة صفوف جديدة INSERT INTO table VALUES (...);
SELECT استرجاع البيانات SELECT columns FROM table WHERE ...;
UPDATE تعديل البيانات UPDATE table SET col=val WHERE ...;
DELETE إزالة صفوف DELETE FROM table WHERE ...;
LOCK التحكم في تزامن الجداول LOCK TABLE table IN lock_mode;
CALL استدعاء برنامج فرعي PL/SQL أو Java CALL procedure_name(arguments);
EXPLAIN PLAN وصف مسار الوصول للبيانات EXPLAIN PLAN FOR SELECT * FROM table;

7. لغة التحكم في البيانات (DCL)

تعريف: DCL

لغة التحكم في البيانات (DCL) تتضمن أوامر مثل GRANT و REVOKE التي تتعامل بشكل رئيسي مع الحقوق والصلاحيات وعناصر التحكم الأخرى لنظام قاعدة البيانات. تُستخدم هذه الأوامر للتحكم في الوصول إلى البيانات عن طريق منح أو سحب الصلاحيات.

✅

GRANT - منح

يمنح صلاحيات محددة للمستخدمين، مما يسمح لهم بتنفيذ عمليات معينة على كائنات قاعدة البيانات

🚫

REVOKE - سحب

يزيل الصلاحيات الممنوحة مسبقاً من المستخدمين، مما يقيد وصولهم إلى كائنات قاعدة البيانات

لماذا DCL مهمة

في قواعد البيانات الحقيقية، لا ينبغي أن يكون لكل مستخدم وصول كامل. تسمح DCL لمسؤولي قواعد البيانات بتطبيق مبدأ الحد الأدنى من الصلاحيات — يحصل المستخدمون فقط على الصلاحيات التي يحتاجونها لأداء عملهم. هذا ضروري لأمان قاعدة البيانات.

8. أمر GRANT - منح الصلاحيات

أمر GRANT يمنح صلاحيات محددة للمستخدمين أو الأدوار على كائنات قاعدة البيانات.

الصيغة العامة

GRANT permission1, permission2, ... ON object TO user;

الصلاحيات الشائعة

الصلاحية الوصف
SELECT صلاحية قراءة/استعلام البيانات من الجدول
INSERT صلاحية إضافة صفوف جديدة إلى الجدول
UPDATE صلاحية تعديل البيانات الموجودة في الجدول
DELETE صلاحية إزالة صفوف من الجدول
ALL PRIVILEGES جميع الصلاحيات المتاحة على الكائن

أمثلة على GRANT

-- منح صلاحية SELECT على جدول Student للمستخدم 'ahmed' GRANT SELECT ON student TO ahmed; -- منح صلاحيتي INSERT و UPDATE للمستخدم 'sara' GRANT INSERT, UPDATE ON student TO sara; -- منح جميع الصلاحيات على جدول Student للمستخدم 'admin' GRANT ALL PRIVILEGES ON student TO admin;

9. أمر REVOKE - سحب الصلاحيات

أمر REVOKE يزيل الصلاحيات الممنوحة مسبقاً من المستخدمين أو الأدوار.

الصيغة العامة

REVOKE permission1, permission2, ... ON object FROM user;

أمثلة على REVOKE

-- سحب صلاحية UPDATE من المستخدم 'sara' REVOKE UPDATE ON student FROM sara; -- سحب جميع الصلاحيات من المستخدم 'ahmed' REVOKE ALL PRIVILEGES ON student FROM ahmed;

GRANT مقابل REVOKE

فكر في GRANT و REVOKE كعكس بعضهما:

  • GRANT ... TO user — يمنح الصلاحيات
  • REVOKE ... FROM user — يسحب الصلاحيات

لاحظ: GRANT يستخدم TO، بينما REVOKE يستخدم FROM.

10. ملخص فئات لغة SQL

أوامر SQL منظمة في ثلاث فئات رئيسية. إليك نظرة عامة شاملة على ما تعلمناه:

🏗️

DDL — لغة تعريف البيانات

تحدد هيكل قاعدة البيانات: CREATE، ALTER، DROP، RENAME

📊

DML — لغة معالجة البيانات

تعمل مع البيانات: INSERT، SELECT، UPDATE، DELETE

🔐

DCL — لغة التحكم في البيانات

تتحكم في الوصول: GRANT، REVOKE

الفئة الغرض الأوامر غُطيت في
DDL تعريف/تعديل هيكل قاعدة البيانات CREATE، ALTER، DROP، RENAME المحاضرة 2
DML معالجة البيانات في الجداول INSERT، SELECT، UPDATE، DELETE المحاضرتان 3 و 4
DCL التحكم في صلاحيات الوصول GRANT، REVOKE المحاضرة 4

  6. امنح صلاحيتي SELECT و INSERT على جدول Student لمستخدم اسمه 'teacher'
  7. اسحب صلاحية INSERT على جدول Student من المستخدم 'teacher'
  8. اكتب عبارة DELETE لإزالة جميع الصفوف من جدول Student

أسئلة المراجعة

اختبر فهمك

  1. ماذا يحدث إذا نفذت عبارة DELETE بدون جملة WHERE؟
  2. ما الفرق بين DELETE FROM table و DROP TABLE table؟
  3. ما الفرق بين UPDATE بدون WHERE و UPDATE مع WHERE؟
  4. كيف يمكنك تحديث أعمدة متعددة في عبارة UPDATE واحدة؟
  5. ما معنى DCL، وما هما أمراها الرئيسيان؟
  6. ما الفرق بين GRANT و REVOKE؟ أيهما يستخدم TO وأيهما يستخدم FROM؟
  7. سمِّ الفئات الرئيسية الثلاث لـ SQL واذكر الأوامر في كل فئة.
  8. لماذا مبدأ الحد الأدنى من الصلاحيات مهم في أمان قواعد البيانات؟

الملخص والنقاط الرئيسية

ما تعلمناه

  1. DELETE: يزيل الصفوف من الجدول باستخدام شرط WHERE. بدون WHERE، يتم حذف جميع الصفوف لكن هيكل الجدول يبقى.
  2. DELETE مقابل DROP: DELETE يزيل صفوف البيانات فقط؛ DROP يزيل الجدول بالكامل بما في ذلك هيكله.
  3. UPDATE (جميع الصفوف): بدون WHERE، يغير UPDATE كل صف في الجدول. استخدم بحذر.
  4. UPDATE (صفوف محددة): مع WHERE، يتم تعديل الصفوف المطابقة للشرط فقط. يمكنك تحديث أعمدة متعددة في عبارة واحدة.
  5. DCL (لغة التحكم في البيانات): GRANT يمنح الصلاحيات؛ REVOKE يسحبها. ضرورية لأمان قاعدة البيانات.
  6. فئات SQL: DDL تعرف الهيكل، DML تعالج البيانات، DCL تتحكم في صلاحيات الوصول.

نظرة مستقبلية: في المحاضرة القادمة، سنستكشف استعلامات SQL بمزيد من العمق — نتعلم الترتيب باستخدام ORDER BY، وعمليات النصوص باستخدام LIKE، والكلمة المفتاحية DISTINCT لإزالة النتائج المكررة.
